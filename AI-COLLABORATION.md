# AI 共同開発ガイドライン (AI Collaboration Guidelines)

このプロジェクトは、開発者（**kojoma**）と AI アシスタント（**Antigravity**）による温かいペアプログラミングと共同開発によって進化しています。
人間の直感や創造性と、AIの知識と実装スピードを融合し、高い品質と楽しさを両立したコードベースを目指します。

---

## 🤝 共同コミットのルール (Co-authoring)

AIアシスタントと協力してコードを設計・実装し、それを Git に記録する際には、お互いの貢献を尊重し、ペアプログラミングの証を残すために **`Co-authored-by`** トレーラーをコミットメッセージに含めます。

### コミットメッセージの形式
コミットメッセージのタイトルの後、空行を1行空けて、メッセージの最下部に以下のように共同作者の情報を記述します。

```text
feat: add exciting new feature for activity analysis

Some detailed description of the changes made during pair programming.

Co-authored-by: Antigravity <antigravity-ai@google.com>
```

※ GitHub上でこのコミットをプッシュすると、開発者のアバターとAIアシスタントのクレジットが並んで表示されるようになります。

## 🛠️ 私たちの歩み (Milestones)

### 🎉 2026年5月23日：リファクタリング、テスト、CIの自動化
人間とAIの最初の本格的なコラボレーションとして、以下の劇的なリファクタリングと環境構築を行いました。

1.  **API Rate Limit 最適化**: 使用されていなかった API コールを削除し、制限回避率を向上。
2.  **堅牢性向上**: 起動時の環境変数バリデーションと、`None` 安全な型変換・例外キャッチを実装。
3.  **データ表現改善**: ランニングピッチを片足 RPM から両足 SPM (2倍) に補正し、マジックナンバーを定数化。
4.  **モジュール化リファクタリング**: 肥大化していた `main()` を、Client取得、取得、整形、保存の単一責任の関数へ美しく分割。
5.  **単体テスト (`pytest`) 導入**: 外部APIを完全にモック化した、ローカルで瞬時に動く 10 ケースのテスト環境を作成。
6.  **GitHub Actions CI 導入**: PR作成時およびマージ時に、自動で `pytest` が実行される軽量かつ安全なテスト用パイプラインを新規追加。

## 💡 AIと快適にペアプロするための3つの約束

当プロジェクトをさらに発展させる際、AIアシスタントを最大限に活かすためのガイドラインです。

1.  **小さな一歩から始める (Incremental Steps)**
    大きな変更を行う際は、一度にすべてを実行しようとせず、今回のようにフェーズ（Phase 1〜4）に細かく分割して一つずつ実装・コミットを積み上げることで、デバッグが極めて簡単になります。
2.  **ローカル環境を汚さない (venv の徹底)**
    追加のパッケージ（`pytest-mock` など）を導入する際は、システムのグローバルな Python 環境を汚さないよう、必ずプロジェクト直下の仮想環境（`venv`）内に閉じ込めてインストールを行います。
3.  **API通信のモック化を心がける (Mocking API)**
    外部 API (Strava) と通信する処理のテストを書く際は、インターネット接続や認証トークンがなくてもローカルで安全かつ瞬時に検証できるよう、常にモックを使用したテスト設計を行います。
